Exercise 0: Build "MyInputFromKeyboardJOptionPaneProject" sample application

In this exercise, you are going to build "MyInputFromKeyboardJOptionPaneProject" sample application that comes with the hands-on lab zip file.  We are going to use this simple application for exercising debugging and refactoring features of NetBeans.

0. Start NetBeans IDE. 
1. Open MyInputFromKeyboardJOptionPaneProject NetBeans project. 

  • Select File->Open Project (Ctrl+Shift+O). The Open Project dialog box appears.
  • Browse down to <LAB_UNZIPPED_DIRECTORY>/javase_debugging/samples directory. 
    • Windows: If you unzipped the 1052_javase_debugging.zip file under C:\ directory, the directory to which you want to browse down should be C:\javase_debugging\samples.
    • Solaris/Linux: If you unzipped the 1052_javase_debugging.zip file under $HOME directory, the directory to which you want to browse down should be $HOME/javase_debugging/samples.
  • Select MyInputFromKeyboardJOptionPaneProject. 
  • Click Open Project Folder.  The MyInputFromKeyboardJOptionPaneProject project node appears under Projects tab window.

2 Build and run MyInputFromKeyboardJOptionPaneProject project.

  • Right-click MyInputFromKeyboardJOptionPaneProject project and select Run.
  • Enter yourname.
  • Click OK.

  • Observe that that message dialog box appears with your name.

In this exercise, you are going to explore various editing features of NetBeans IDE.

  1. Refactoring
  2. Error stripe

1. Open InputFromKeyboardJOptionPane.java under MyInputFromKeyboardJOptionPaneProject->Source Packages->org.me.hello.
2. Select name string and right-click it.  Select Refactor and then Rename.



3. In the Rename Local Vraible name dialog box, type username in the New Name field. Click Refactor.



4.  Observe the result of refactoring of renaming.


5.  Right-click MyInputFromKeyboardJOptionPaneProject project node and select Run.  Observe that the application behaves the same.

(1.3) Code templates

1. Select Tools from the menu bar.

Exercise 3: Debugging

In this exercise, you will exercise the following features of NetBeans IDE.

    1. Source level JSP page debugging

(3.1) Source level debugging of the JSP page

0. Make sure MyInputFromKeyboardJOptionPaneProject project is the Main project.  (A project node in bold font is the Main project.)  If it is not, right click MyInputFromKeyboardJOptionPaneProject and select Set Main Project.

1. Open public class InputFromKeyboardJOptionPane.java  in the editor window.
2. To make a breakpoint on the String msg = "Hello " + username + "!"; line (at line number 15 below), move your cursor to the location of the gray colored vertical bar of the line  and click.  The dark pink colored bar indicates that the breakpoint is created.


3.  Right click MyInputFromKeyboardJOptionPaneProject project node and select Debug.  NetBeans will rebuild and deploy the application for debugging.


4. When a dialog prompts you to enter your name, enter a name and click OK button.


5. Note that the breakpoint is hit. 
6. Observe that Variables tab window gets displayed in the bottom of the IDE. Observe that the username variable is set with your name, Sang Shin, in the example below.

7. You can change the value of the variables.  Change the value of the username variable to something else, Daniel Shin, in the example below. Make sure you press the Return key after the change so that the change is recorded in the debugging session as shown below.  The dark green bar indicates that the change has been recorded.


7.  Select Debug from the top-level menu bar and select Step Over  (or press F8 key) to step over.


8. Observe that another local variable, msg, is now set with Hello, Daniel Shin.


9. Select Debug from the top-level menu bar and select Continue  (or press F5 key) to continue.


10. Observe that the debugging session is over (since we did not have any more breakpoints).
